このツールでできること

電子名刺セットを自動生成するツールです

必要な情報を入力してボタンを押すだけで、4つのHTMLファイルと画像フォルダをまとめたZIPファイルが生成されます。GitHubまたはXserverにアップするだけで、すぐに使える電子名刺が完成します。

card.html QRコード付き名刺。スマホで見せて使う 自分

準備するもの

アバター画像は必須です。名前・肩書きとあわせて、この3点を用意してから作業を始めてください。
項目必須推奨サイズ・形式備考
アバター画像 必須 正方形・400px以上 顔写真・アイコン等

5

集客ルート設定

名刺を受け取った相手への導線を4つのルートから設定できます。それぞれlanding.html（名刺ページ）とprofile.html（プロフィールページ）に個別にON/OFFで設定可能です。

ルート A
LINE公式
LINEリストへ誘導。リードマグネット（特典画像＋テキスト）も設定可
ルート B
Web Push
プロ版のみ対応
ルート C
メルマガ
メルマガ登録ページへ誘導。ボタンラベルとリードマグネットを設定可
ルート D
ブックマーク
URLコピーボタンとホーム画面追加案内を表示
ルートAのLINE URLは「SNS欄から引用」ボタンでSNS欄に入力済みのURLを自動入力できます。2回入力する必要はありません。
リードマグネットとは

ルートA（LINE）とルートC（メルマガ）には、登録を促す特典情報を表示できます。特典テキストと特典画像（推奨：1200x630px）を設定すると、profile.htmlのボタン上部に表示されます。

リードマグネットの画像・テキストはprofile.htmlのみに表示されます。landing.html（名刺ページ）はシンプルさを保つためボタンのみ表示されます。

告知エリアの設定

キャンペーンやお知らせをlanding.html・profile.htmlに表示できます。複数登録でき、表示期間を日付で制御できます。

項目必須説明
ラベル必須「期間限定」「新着」等の短いタグ
告知テキスト必須告知の本文
リンクURL任意「詳しくはこちら」ボタンのリンク先
ボタンテキスト任意ボタンのラベル（デフォルト：詳しくはこちら）
画像任意profile.htmlのカード上部に表示
表示開始日任意この日より前は非表示
表示終了日任意この日を過ぎると自動で非表示
「表示開始日」はその告知を表示し始める日付です。キャンペーン開始日と同じかそれ以前の日付を設定してください。設定した日付より前だと表示されません。
告知は複数追加できます。告知が不要な場合は追加しないか、テキストを空にしてください。テキストが空の告知は自動的に除外されます。

7

ZIPを生成してアップロードする

1
OGP画像の設定（任意）
SNSでシェアされたときのサムネイル画像です。アップロードしない場合は名刺デザインが自動で画像化されます。
2
「ZIPを生成してダウンロード」をクリック
必須項目（アバター・名前・肩書き）が入力済みであることを確認してからクリックしてください。
3
ZIPを展開する
ダウンロードされたZIPファイルを展開します。フォルダ構成はそのままにしてください。
4
フォルダごとアップロードする
HTMLファイルとimagesフォルダを同じ階層のままGitHub PagesまたはXserverにアップロードします。card.htmlはアップロード不要です。
アップロードする構成
├── landing.html ← QRの読み取り先
├── profile.html ← リンクまとめページ
├── about.html ← 作成した場合のみ
└── images/ ← 必ずフォルダごと
card.html はアップロード不要（スマホで使うため）
GitHub Pagesでの公開手順
1
GitHubにリポジトリを作成
リポジトリ名は半角英数字で（例：my-card）。Publicに設定。
2
ファイルをアップロード
「Add file」→「Upload files」で、展開したフォルダの中身（landing.html・profile.html・images/フォルダ）をすべて選択してアップロード。
3
GitHub Pagesを有効化
Settings → Pages → Branch を「main」に変更して「Save」をクリック。数分後に公開されます。
4
公開URLを確認
https://ユーザー名.github.io/リポジトリ名/landing.html がlanding.htmlのURLになります。

QRコード名刺の使い方

landing.htmlの公開URLが確定したら、QRコード入りの名刺を生成します。

1
ツールに戻り、landing.htmlのURLを入力
「QRコード付きcard.htmlを再生成する」欄に、公開後に確定したURLを入力します。
2
card.htmlまたはPNG画像を生成
「QR付きcard.htmlを再生成」ボタンでHTMLファイル、「QR付き名刺画像（PNG）を生成」ボタンで画像ファイルをダウンロードできます。
3
スマホに保存する
生成したcard.htmlまたはPNG画像をスマホに転送・保存します。サーバーへのアップロードは不要です。
4
名刺交換で使う
名刺交換の際にスマホ画面を相手に見せて、QRコードを読み取ってもらいます。相手のスマホでlanding.htmlが開きます。
スマホのホーム画面にcard.htmlを追加しておくと、アプリのようにすぐ開けて便利です。iPhoneの場合はSafariで開いて「共有」→「ホーム画面に追加」、Androidは「メニュー」→「ホーム画面に追加」で追加できます。
内容を変更したいときは

プロフィール情報や告知を更新したい場合は、ツールで修正してからZIPを再生成し、フォルダごとアップロードし直してください。landing.htmlのURLが変わらない限り、QRコードの再生成は不要です。

よくある質問

ブラウザのlocalStorageに保存されます。同じブラウザで開けば次回も入力済みの状態で再開できます。別のブラウザや別のデバイスには引き継がれません。
「表示開始日」の設定を確認してください。設定した日付より前だと自動的に非表示になります。表示開始日は告知を表示し始める日付です。今日以前の日付を設定するか、空白にすると即時表示されます。
card.htmlはimages/フォルダと同じ場所に置かないと画像が表示されません。ローカルで確認する場合はZIPを展開したフォルダごと開いてください。スマホで使う場合はサーバーにアップ後の確認は不要です。
landing.htmlのURLをツールに入力した状態でZIPを生成しているか確認してください。URLを入力せずに生成するとog:imageが相対パスになるため、SNSでは表示されません。また、XのCard Validatorにかけるとキャッシュがリフレッシュされます。
Settings → Pages でGitHub Pagesが有効になっているか確認してください。有効化後も反映に数分かかる場合があります。ブランチが「main」に設定されていることも確認してください。
Formspree（formspree.io）に無料登録してフォームを作成し、発行されるエンドポイントURL（https://formspree.io/f/xxxxxxxx）をツールの「問い合わせフォーム」欄に入力してください。無料プランは月50件まで受信できます。
はい。ツールのURLを共有するだけで、別の人が自分の名刺を作れます。入力データは各自のブラウザに保存されるため、データが混在することはありません。
プロ版では内容のリアルタイム更新（再アップ不要）、Web Push通知によるリスト構築、GAS管理画面、購読者数の確認ができます。ライト版では内容を変更するたびにZIPを再生成してアップロードし直す必要があります。
